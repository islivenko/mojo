# План работ по задаче "BI-аналитика для Mojo Agency"

Этот документ описывает шаги для реализации задачи по построению BI-отчетности для анализа ключевых показателей Mojo Agency.

**Технологический стек:**
*   **SQL диалект:** Trino
*   **Система визуализации:** Superset

## Фаза 1: Подготовка SQL-запросов для KPI

На основе данных из Bitrix24 мы подготовим SQL-запросы для расчета ключевых показателей эффективности (KPI). Основные источники данных:
*   `bitrix24.crm_deal`
*   `bitrix24.crm_deal_stage_history`
*   `bitrix24.telephony_call`
*   Другие релевантные таблицы

SQL-запросы для каждого KPI будут храниться в отдельных файлах.

### Стандарт именования алиасов (псевдонимов)

Для обеспечения совместимости с BI-системами и поддержания читаемости кода, все алиасы в SQL-запросах должны соответствовать следующим правилам:
*   **Формат:** `Latin_Snake_Case` (слова разделяются нижним подчеркиванием).
*   **Символы:** Использовать только латинские буквы, цифры и нижнее подчеркивание.
*   **Пробелы:** Пробелы в названиях заменяются на `_`.
*   **Кавычки:** Алиасы задаются без использования кавычек.
*   **Транслитерация:** Названия на других языках (например, польском) транслитерируются на латиницу.

**Пример:**
```sql
-- Правильно
SELECT uf_crm_8_1759679282737 AS Zgloszone_kadry_T_N
-- Неправильно
SELECT uf_crm_8_1759679282737 AS "Zgłoszone kadry T/N"
```

## Фаза 2: Визуализация

Подготовленные SQL-запросы будут использованы для создания дашбордов в **Superset**.
